## 1. Overview & Principles

### 1.1 Product Vision
An offline-first desktop application that enables Indonesian rural cooperatives (KDKMP) to manage operations, perform SAK EP-compliant financial analysis, and aggregate data hierarchically from Desa → Kabupaten → Provinsi → Nasional.

### 1.2 Design Principles
| Principle | Description |
|---|---|
| **Offline-first** | Every feature works without internet. Sync is secondary. |
| **Low-literacy UX** | Large fonts, simple Indonesian labels, minimal jargon, visual icons. |
| **Desktop-class compute** | Heavy analysis (Monte Carlo, sensitivity) runs locally — no server round-trip. |
| **Data sovereignty** | Member data never leaves the local machine unless explicitly synced. |
| **Progressive disclosure** | Show simple view first; advanced features behind a "Lanjutan" toggle. |

### 1.3 Tech Assumptions
- **Desktop shell:** Tauri 2.x
- **Local DB:** SQLite (via rusqlite)
- **Frontend:** SvelteKit (or React) + Tailwind CSS
- **Charts:** D3.js or Chart.js
- **PDF:** Native Tauri print API
- **Sync:** Custom REST-like pull/push over HTTPS

## 3. User Roles & Permissions

| Role | Description | Key Permissions |
|---|---|---|
| **Pengurus** (Admin) | Cooperative manager / ketua | Full access: accounting, analysis, sync, reports, settings |
| **Operator** (Staff) | Daily data entry staff | Members, journal entries, view reports (no delete, no settings) |
| **Pengawas** (Auditor) | Supervisor / auditor | Read-only: all reports, audit trail, dashboard |
| **Sync Peer** (Machine) | Upstream Kab/Prov node | Sync API only — no interactive UI |

## 4. Application Screens

### 4.1 Splash / Loading

**Purpose:** App startup while SQLite initializes and data loads.

**UI Layout:**
```
┌──────────────────────────────────┐
│                                  │
│          [Logo KDKMP]            │
│                                  │
│     Sistem Informasi KDKMP       │
│                                  │
│   [Loading spinner / progress]   │
│                                  │
│   "Memuat data lokal..."         │
│                                  │
│   v1.0.0                         │
└──────────────────────────────────┘
```

**Behavior:**
- Show immediately on launch
- Duration: ~500ms–2s (SQLite init + schema migration)
- If DB migration fails: show error screen with "Hubungi admin" + error code
- Auto-transitions to Login screen on success

**States:**
| State | Display |
|---|---|
| Loading | Logo + spinner + "Memuat data lokal..." |
| Migration in progress | "Memperbarui struktur data..." + indeterminate progress |
| Error | Icon error + "Gagal memuat database. Kode: ERR-001" + "Coba Lagi" button |
| Slow startup (>5s) | "Bila pertama kali, ini normal. Harap tunggu..." |

### 4.2 Local PIN Login

**Purpose:** Authenticate user to the local application. No network required.

**UI Layout:**
```
┌──────────────────────────────────┐
│          [Back arrow]            │
│                                  │
│          [Logo kecil]            │
│                                  │
│     Masukkan PIN Anda            │
│                                  │
│     ┌──────────────────────┐     │
│     │  ●  ●  ●  ●  ●  ●   │     │
│     └──────────────────────┘     │
│          (6 digit)               │
│                                  │
│     [1] [2] [3]                  │
│     [4] [5] [6]                  │
│     [7] [8] [9]                  │
│          [0]  [⌫]               │
│                                  │
│     ───────────────────────      │
│     Login dengan biometric       │
│     (jika tersedia)              │
└──────────────────────────────────┘
```

**Interaction:**
1. On first launch: show "Buat PIN Baru" screen (enter PIN twice + recovery question)
2. Subsequent launches: show PIN entry
3. Tap number buttons to enter PIN (dots fill as digits entered)
4. After 6 digits: auto-validate
5. On success: transition to Main Dashboard
6. On failure: shake animation, clear dots, show "PIN salah. Sisa percobaan: N"
7. After 5 failures: lockout for 60s with countdown timer
8. "Lupa PIN?" link → show recovery answer flow

**States:**
| State | Display |
|---|---|
| First launch | "Buat PIN Baru" — enter twice + recovery question |
| Normal login | PIN pad — "Masukkan PIN Anda" |
| Wrong PIN | Shake + "PIN salah" + remaining attempts |
| Locked out | "Terlalu banyak percobaan. Tunggu 60 detik." + countdown |
| Recovery | Recovery question text input → reset PIN |

**Sync Protocol (for backend agent reference):**
1. **Authentication:** Each node has a unique ID + API token
2. **Push (Upload):** POST `/api/v1/sync/push` with JSON body containing changed entities since `lastSyncAt`
3. **Pull (Download):** GET `/api/v1/sync/pull?since={lastSyncAt}&nodeId={nodeId}`
4. **Conflict resolution:** Last-write-wins per entity, based on `updatedAt` timestamp
5. **Audit:** All sync operations logged in `sync_audit` table
6. **Retry:** Exponential backoff (30s, 2min, 5min, 15min) on failure
7. **Entities synced:** Members, Journal Entries, COA changes, Profile updates

**Interactions:**
- [SINKRONKAN SEKARANG] → show progress modal with step-by-step status
- Progress modal: "Mengupload data..." → "Mendownload data..." → "Selesai!" with checkmark
- [UJI KONEKSI] → ping remote node, show latency + "Terhubung" / "Gagal"
- Auto-sync runs in background; status updates on dashboard
- Offline: entire section shows 🟡 "Tidak ada koneksi internet" with muted controls

**Edge Cases:**
- Conflict during sync: "Data [entity] memiliki konflik. Menggunakan versi terbaru."
- Remote node unreachable: "Node induk tidak dapat dijangkau. Coba lagi nanti."
- Large payload: show progress percentage + "Mengupload 45 dari 120 entri..."
- First sync: "Sinkronisasi pertama mungkin memakan waktu lama"

    locked_until TEXT,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (cooperative_id) REFERENCES cooperatives(id)
);

-- Members
CREATE TABLE members (
    id TEXT PRIMARY KEY,
    cooperative_id TEXT NOT NULL,
    nik TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    place_of_birth TEXT,
    date_of_birth TEXT,
    gender TEXT CHECK(gender IN ('L','P')),
    occupation TEXT,
    education TEXT,
    rt TEXT,
    rw TEXT,

    deactivated_at TEXT,
    updated_at TEXT DEFAULT (datetime('now')),
    sync_status TEXT DEFAULT 'pending',
    FOREIGN KEY (cooperative_id) REFERENCES cooperatives(id)
);

-- Chart of Accounts (SAK EP)
CREATE TABLE coa_accounts (
    code TEXT PRIMARY KEY,
    cooperative_id TEXT NOT NULL,
    name TEXT NOT NULL,
    type TEXT NOT NULL CHECK(type IN ('aset','kewajiban','ekuitas','pendapatan','beban')),
    category TEXT,
    normal_balance TEXT NOT NULL CHECK(normal_balance IN ('debit','kredit')),
    balance REAL DEFAULT 0,
    is_active INTEGER DEFAULT 1,
    parent_code TEXT,
    sort_order INTEGER,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (cooperative_id) REFERENCES cooperatives(id)
);

    tags TEXT, -- JSON array
    created_by TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now')),
    sync_status TEXT DEFAULT 'pending',

    FOREIGN KEY (created_by) REFERENCES local_users(id)

-- Journal Lines

    id TEXT PRIMARY KEY,
    journal_entry_id TEXT NOT NULL,
    account_code TEXT NOT NULL,
    description TEXT,
    debit REAL DEFAULT 0,

    FOREIGN KEY (account_code) REFERENCES coa_accounts(code)
);

    eirr REAL,
    ebcr REAL,
    tier INTEGER,
    calculated_at TEXT DEFAULT (datetime('now')),

## 9. Interaction Patterns

### 9.1 Navigation
- **Stack navigation** within each tab (back button always visible)
- **Bottom tab** switches between 3 main sections
- **Deep links** not required (offline-first desktop app)
- **Keyboard shortcuts** (future): `Ctrl+N` new journal, `Ctrl+S` save, `Ctrl+P` print

### 9.2 Data Entry
- **Auto-save** is NOT used — explicit Save/Cancel on all forms
- **Confirmation on destructive actions**: delete, bulk operations, data reset
- **Validation on blur**: inline validation triggers when field loses focus
- **NIK auto-format**: groups of 4 digits with space (e.g., "3512 3456 7890 123")
- **Rupiah formatting**: as user types, auto-add thousand separators
- **Date picker**: calendar widget with month/year navigation, keyboard input also accepted (DD-MM-YYYY)

### 9.3 Data Display
- **Paginated tables**: 10 rows default, "Muat lebih banyak" or page numbers
- **Infinite scroll on mobile-like views**: member list, journal list
- **Sortable columns**: tap column header to sort ascending/descending (indicator arrow)
- **Filterable**: search bar + dropdown filters on list screens
- **Export to Excel**: all tables have export option (CSV minimum)

### 9.4 Charts & Visualization
- **RAG Indicators**: always use both color AND icon/pattern (accessibility)
- **Mini charts**: on dashboard, 6-month trend sparklines
- **Full charts**: on Financial Analysis / Sensitivity, full-size responsive charts
- **Tooltip on hover/click**: exact values in tooltip
- **Chart download**: long-press or right-click → "Simpan sebagai gambar"

### 9.5 Sync
- **Non-blocking**: sync runs in background thread. User continues working.
- **Progress indication**: subtle progress bar in header during sync
- **Conflict resolution**: automatic (last-write-wins). Logged in audit trail.
- **Manual trigger**: explicit "Sinkronkan Sekarang" button for power users
- **Offline queue**: all creates/updates go to `sync_status = 'pending'` queue

### 9.6 Accessibility
- **Font size**: minimum 16px body, 20px headings
- **Color independence**: RAG uses icon/shape in addition to color
- **Focus indicators**: visible outline on all interactive elements
- **Keyboard navigation**: Tab through form fields, Enter to submit
- **Screen reader labels**: all icons have `aria-label`

### 9.7 Responsive / Window Resize
- **Minimum window size**: 1024×768 (desktop only — no mobile in MVP)
- **Default window size**: 1280×800
- **Layout**: fluid grid that adapts to resize (cards reflow, but no breakpoints)
- **Scroll**: vertical scrolling on content areas; header and bottom nav fixed

## Appendix B: Computation Formulas

```
ENPV = Σ (CFt / (1 + r)^t) - Initial Investment
Where: CFt = Cash flow at year t, r = discount rate, t = year (1..n)

EIRR = r where Σ (CFt / (1 + r)^t) = Initial Investment
(Solved via Newton-Raphson or binary search)

EBCR = Σ (CFt / (1 + r)^t) / Initial Investment
```

**Tiering Logic:**
```
if ENPV > 0 AND EIRR > discountRate AND EBCR >= 1.0 → Tier 1 (LAYAK)
if ENPV > 0 AND (EIRR > discountRate OR EBCR >= 1.0) → Tier 2 (CUKUP)
else → Tier 3 (TIDAK LAYAK)
```
